\begin{abstract}

Functional reactive programming (FRP) makes it possible to express temporal
aspects of computations in a declarative way. Recently we developed two kinds of
categorical models of FRP: abstract process categories (APCs) and concrete
process categories (CPCs). Furthermore we showed that APCs generalize CPCs. In
this paper, we extend APCs with additional structure. This structure models
recursion and corecursion operators that are related to time. We show that the
resulting categorical models generalize those CPCs that impose an additional
constraint on time scales. This constraint boils down to ruling out
$œâ$-supertasks, which are closely related to Zeno‚Äôs paradox of Achilles and the
tortoise.

\end{abstract}



\extsection{introduction}{Introduction}

Functional reactive programming (FRP) makes it possible to express temporal
aspects of computations in a declarative way. Traditional FRP is based on
behaviors and events, which denote time-varying values and values attached to
times, respectively. There is a Curry‚ÄìHoward correspondence between traditional
FRP and an intuitionistic temporal logic with ‚Äúalways‚Äù and ‚Äúeventually‚Äù
modalities~\cite{jeltsch:entcs-286,jeffrey:plpv-2012}. Thereby the type
constructor for behaviors corresponds to ‚Äúalways,‚Äù and the type constructor for
events corresponds to ‚Äúeventually.‚Äù

Extending the temporal logic with ‚Äúuntil‚Äù operators leads to an extended variant
of FRP. Thereby proofs of ‚Äúuntil‚Äù propositions correspond to a new class of FRP
constructs, which we call processes. Processes in the FRP sense combine
continuous and discrete aspects and generalize behaviors and events
\cite[Section~2]{jeltsch:plpv-2013}. We give an introduction to FRP with
processes in \sectionref{functional-reactive-programming-with-processes}.

We have developed two kinds of models of FRP with processes, which can also be
used to model temporal logic with ‚Äúuntil:‚Äù
\begin{description}

\item[Abstract process categories (APCs)~\cite{jeltsch:plpv-2014}]

are defined purely axiomatically. They are an extension of temporal
categories~\cite{jeltsch:entcs-286}, which in turn build on categorical models
of intuitionistic S4~\cite{kobayashi:tcs-175-1,bierman:sl-65-3}.

\item[Concrete process categories (CPCs)~\cite{jeltsch:plpv-2013}]

are not defined in a purely axiomatic manner, but use concrete constructions to
express time-dependence of type inhabitance and causality of FRP operations.

\end{description}
Abstract process categories generalize concrete process categories. We describe
APCs in \sectionref{abstract-process-categories} and CPCs in
\sectionref{concrete-process-categories}.

The aim of this paper is to extend APCs and CPCs in order to model recursion and
corecursion on processes. We make the following contributions:
\begin{itemize}

\item

In \sectionref{apcs-with-recursion-and-corecursion}, we develop APCs with
recursion and corecursion (‚Ñõ-APCs). ‚Ñõ-APCs extend APCs with recursive comonad
and completely iterative monad structures that model recursion and corecursion
on processes. The extensions to APCs arise naturally as extensions of ideal
comonad and ideal monad structures that APCs already contain.

\item

In \sectionref{cpcs-with-recursion-and-corecursion}, we develop CPCs with
recursion and corecursion (‚Ñõ-CPCs). ‚Ñõ-CPCs differ from CPCs in that they impose
an additional constraint on time scales. This constraint boils down to ruling
out $œâ$-supertasks~\cite{laraudogoitia:supertasks}. We show that ‚Ñõ-APCs
generalize ‚Ñõ-CPCs.

\end{itemize}

We discuss related work in \sectionref{related-work} and give conclusions and an
outlook on further work in \sectionref{conclusions-and-further-work}.

\extsection{functional-reactive-programming-with-processes}
           {Functional Reactive Programming with Processes}

The FRP dialect we consider here is based on a linear notion of time. However
the time scale does not need to be discrete; so the time domain can be any
totally ordered set. Type inhabitance is time-dependent, which means that every
FRP type corresponds to a time-indexed family of sets of inhabitants.

Besides the ordinary type constructors $1$, $0$, $√ó$, $+$, and~$‚Üí$, our FRP
dialect has two binary type constructors $‚ñ∑‚Ä≥‚ÇÄ$ and~$‚ñ∑‚Ä≥‚ÇÅ$, which we call the
strong and the weak basic process type constructor. Both have higher precedence
than the other binary type constructors.

A value that inhabits a type $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÄ œÑ‚ÇÇ$ at a time~$t$ corresponds to a tuple
with the following elements:
\begin{itemize}

\item

a time~$t‚Ä≤ > t$

\item

a function~$h$ that maps each time $t‚Ä≥$ with $t < t‚Ä≥ < t‚Ä≤$ to a value that
inhabits $œÑ‚ÇÅ$ at~$t‚Ä≥$

\item

a value~$y$ that inhabits $œÑ‚ÇÇ$ at~$t‚Ä≤$

\end{itemize}
The function~$h$ denotes a time-varying value that exists between times $t$
and~$t‚Ä≤$. We call this time-varying value the continuous part of the process.
The pair $(t‚Ä≤, y)$ denotes an event that occurs at time~$t‚Ä≤$ and carries the
value~$y$. We call this event the terminal event of the process.

A value that inhabits a type $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÅ œÑ‚ÇÇ$ at a time~$t$ is either a process that
inhabits $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÄ œÑ‚ÇÇ$ at~$t$ or a time-varying value of type~$œÑ‚ÇÅ$ that starts
immediately after~$t$ and persists forever. We regard constructs of the latter
kind as special processes that never terminate and thus have an infinite
continuous part and no terminal event.


An inhabitant of a type $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÄ œÑ‚ÇÇ$ or $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÅ œÑ‚ÇÇ$ has a continuous part that
assigns values only to future times. We define type constructors $‚ñ∑‚Ä≤‚ÇÄ$ and~$‚ñ∑‚Ä≤‚ÇÅ$
for processes that start at the present time:
\begin{align}
œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÄ œÑ‚ÇÇ & = œÑ‚ÇÅ √ó œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÄ œÑ‚ÇÇ & œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÅ œÑ‚ÇÇ & = œÑ‚ÇÅ √ó œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÅ œÑ‚ÇÇ
\end{align}
A pair $(x, p)$ represents a process that starts with~$x$ as the initial value
of its continuous part, and continues like the process~$p$.

An inhabitant of a type $œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÄ œÑ‚ÇÇ$ or $œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÅ œÑ‚ÇÇ$ can only terminate in the
future. We define type constructors $‚ñ∑‚ÇÄ$ and~$‚ñ∑‚ÇÅ$ for processes that may
terminate at the present time:
\begin{align}
œÑ‚ÇÅ ‚ñ∑‚ÇÄ œÑ‚ÇÇ & = œÑ‚ÇÇ + œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÄ œÑ‚ÇÇ & œÑ‚ÇÅ ‚ñ∑‚ÇÅ œÑ‚ÇÇ & = œÑ‚ÇÇ + œÑ‚ÇÅ ‚ñ∑‚Ä≤‚ÇÅ œÑ‚ÇÇ
\end{align}
A value $Œπ‚ÇÅ(y)$ represents a process whose terminal event occurs immediately and
carries the value~$y$, while a value $Œπ‚ÇÇ(p)$ represents the process~$p$, which
does not terminate immediately.

From the process type constructors, we derive the type constructors $‚ñ°‚Ä≤$, $‚ñ°$,
$‚óá‚Ä≤$, and~$‚óá$ as follows:
\begin{align}
‚ñ°‚Ä≤œÑ & = œÑ ‚ñ∑‚Ä≥‚ÇÅ 0 & ‚ñ°œÑ & = œÑ ‚ñ∑‚Ä≤‚ÇÅ 0 \\
‚óá‚Ä≤œÑ & = 1 ‚ñ∑‚Ä≤‚ÇÄ œÑ & ‚óáœÑ & = 1 ‚ñ∑‚ÇÄ œÑ
\end{align}
Inhabitants of types $‚ñ°‚Ä≤œÑ$ and $‚ñ°œÑ$ are non-terminating time-varying values and
are called behaviors. Inhabitants of types $‚óá‚Ä≤œÑ$ and $‚óáœÑ$ are events.

The FRP dialect described above corresponds to an intuitionistic temporal logic
via a Curry‚ÄìHoward isomorphism. Thereby time-dependent type inhabitance is
related to time-dependent trueness of temporal propositions. The type
constructors $‚ñ∑‚ÇÄ$ and~$‚ñ∑‚ÇÅ$ correspond to the strong and weak ‚Äúuntil‚Äù operators
$ùí∞$ and~$ùí≤$ from linear-time temporal logic (LTL), and the type constructors $‚ñ°$
and~$‚óá$ correspond to the ‚Äúalways‚Äù and ‚Äúeventually‚Äù modalities.

\extsection{abstract-process-categories}{Abstract Process Categories}

Abstract process categories (APCs) are axiomatically defined categorical models
of FRP with processes, which we developed recently~\cite{jeltsch:plpv-2014}.
They are an extension of temporal categories, which are models of FRP with
behaviors and events, but without processes~\cite{jeltsch:entcs-286}. In this
section, we give an introduction to APCs.

\extsubsection{the-basics}{The Basics}

An APC is a category~$ùíû$ with some additional structure. FRP types are modeled
by objects of~$ùíû$. If objects $A$ and~$B$ model FRP types $œÑ‚ÇÅ$ and~$œÑ‚ÇÇ$, the
morphisms from~$A$ to~$B$ model operations that turn every value that
inhabits~$œÑ‚ÇÅ$ at some time into a value that inhabits~$œÑ‚ÇÇ$ at the same time.

Since FRP has the usual type constructors for finite products, finite sums, and
function spaces, we require~$ùíû$ to be a cartesian closed category (CCC) with
finite coproducts.

\extsubsection{temporal-functors}{Temporal Functors}

Let $ùüê$ denote the interval category, that is, the category with exactly two
objects $0$ and~$1$ and a single non-identity morphism $w : 0 ‚Üí 1$. Let
furthermore $ùí¨$ be the category $ùüê √ó ùíû √ó ùíû$. We require the existence of a
functor $‚ñ∑‚Ä≥ : ùí¨ ‚Üí ùíû$, which we call the basic temporal functor. We use the
notation $A ‚ñ∑‚Ä≥_W B$ for $‚ñ∑‚Ä≥(W, A, B)$. We model the process type constructors
$‚ñ∑‚Ä≥‚ÇÄ$ and~$‚ñ∑‚Ä≥‚ÇÅ$ by the partial functor applications $‚ñ∑‚Ä≥‚ÇÄ$ and~$‚ñ∑‚Ä≥‚ÇÅ$, which are
themselves functors from $ùíû √ó ùíû$ to~$ùíû$.

Every inhabitant of a type $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÄ œÑ‚ÇÇ$ corresponds to an inhabitant of $œÑ‚ÇÅ ‚ñ∑‚Ä≥‚ÇÅ
œÑ‚ÇÇ$. So there should be an operation that performs a type conversion from $‚ñ∑‚Ä≥‚ÇÄ$
to~$‚ñ∑‚Ä≥‚ÇÅ$. We use the natural transformation $‚ñ∑‚Ä≥_w : ‚ñ∑‚Ä≥‚ÇÄ ‚Üí ‚ñ∑‚Ä≥‚ÇÅ$ to model this
operation.

From the functor~$‚ñ∑‚Ä≥$, we derive functors $‚ñ∑‚Ä≤, ‚ñ∑ : ùí¨ ‚Üí ùíû$ that model the type
constructors $‚ñ∑‚Ä≤‚ÇÄ$, $‚ñ∑‚Ä≤‚ÇÅ$, $‚ñ∑‚ÇÄ$, and~$‚ñ∑‚ÇÅ$ as well as functors $‚ñ°‚Ä≤, ‚ñ°, ‚óá‚Ä≤, ‚óá : ùíû
‚Üí ùíû$ that model the type constructors for behaviors and events:
\begin{align}
\label{equation:derived-process-functors}
A ‚ñ∑‚Ä≤_W B & = A √ó A ‚ñ∑‚Ä≥_W B & A ‚ñ∑_W B  & = B + A ‚ñ∑‚Ä≤_W B \\
‚ñ°‚Ä≤A & = A ‚ñ∑‚Ä≥‚ÇÅ 0 & ‚ñ°A & = A ‚ñ∑‚Ä≤‚ÇÅ 0                      \\
\label{equation:event-functors}
‚óá‚Ä≤B & = 1 ‚ñ∑‚Ä≤‚ÇÄ B & ‚óáB & = 1 ‚ñ∑‚ÇÄ B
\end{align}
We call the functors introduced in \eqref{equation:derived-process-functors}
through~\eqref{equation:event-functors} derived temporal functors.

APCs do not necessarily use the category~$ùüê$ to model how weak a process type
constructor is. Any category~$ùí≤$ with finite products is allowed instead. In
particular, we can use any category that corresponds to a partially ordered set
with finite meets. This makes it possible to model more advanced termination
properties. An example of such a property is termination with an upper bound on
the termination time. We discuss this example in
\subsectionref{the-basic-temporal-functor}.

\extsubsection{process-expansion-and-joining}{Process Expansion and Joining}

An APC contains a natural transformation~$Œ∏‚Ä≥$ with
\begin{equation*}
Œ∏‚Ä≥_{W, A, B} : A ‚ñ∑‚Ä≥_W B ‚Üí \left(A ‚ñ∑‚Ä≤_W B\right) ‚ñ∑‚Ä≥_W B\enspace.
\end{equation*}
This natural transformation models an FRP operation that turns a process~$p$
into a process~$p‚Ä≤$ such that the following holds:
\begin{itemize}

\item

The process~$p‚Ä≤$ terminates if and only if~$p$ terminates.

\item

The terminal event of~$p‚Ä≤$, if any, is the terminal event of~$p$.

\item

The value of the continuous part of~$p‚Ä≤$ at a time~$t$ is the suffix of~$p$ that
starts at~$t$.

\end{itemize}
We call this FRP operation process expansion. We derive variants of~$Œ∏‚Ä≥$ that
work with $‚ñ∑‚Ä≤$ and~$‚ñ∑$ instead of~$‚ñ∑‚Ä≥$:
\begin{align}
Œ∏‚Ä≤_{W, A, B} & \relwithsizeof=: A ‚ñ∑‚Ä≤_W B ‚Üí \left(A ‚ñ∑‚Ä≤_W B\right) ‚ñ∑‚Ä≤_W B                    \notag\\
Œ∏‚Ä≤_{W, A, B} & =                \left\langle\id_{A ‚ñ∑‚Ä≤_W B}, Œ∏‚Ä≥_{W, A, B} ‚àò œÄ‚ÇÇ\right\rangle \\[\topsep]
Œ∏_{W, A, B}  & \relwithsizeof=: A ‚ñ∑_W B ‚Üí \left(A ‚ñ∑‚Ä≤_W B\right) ‚ñ∑_W B                      \notag\\
\label{equation:no-prime-theta}
Œ∏_{W, A, B}  & =                \id_B + Œ∏‚Ä≤_{W, A, B}
\end{align}

Besides the natural transformation~$Œ∏‚Ä≥$, an APC also contains a natural
transformation~$œë‚Ä≥$ with
\begin{equation*}
œë‚Ä≥_{W, A, B} : A ‚ñ∑‚Ä≥_W (A ‚ñ∑_W B) ‚Üí A ‚ñ∑‚Ä≥_W B\enspace.
\end{equation*}
This natural transformation models an FRP operation that turns a process~$p$
into a process~$p‚Ä≤$ such that the following holds:
\begin{itemize}

\item

If $p$ terminates, and its terminal event carries a process~$p^*$, then $p‚Ä≤$ is
the result of concatenating the continuous part of~$p$ and the process~$p^*$.

\item

If $p$ does not terminate, then $p‚Ä≤$ does not terminate, and the continuous part
of~$p‚Ä≤$ is the continuous part of~$p$.

\end{itemize}
We call this FRP operation process joining. We derive variants of~$œë‚Ä≥$ that work
with $‚ñ∑‚Ä≤$ and~$‚ñ∑$ instead of~$‚ñ∑‚Ä≥$:
\begin{align}
œë‚Ä≤_{W, A, B} & \relwithsizeof=: A ‚ñ∑‚Ä≤_W (A ‚ñ∑_W B) ‚Üí A ‚ñ∑‚Ä≤_W B                   \notag\\
\label{equation:single-prime-vartheta}
œë‚Ä≤_{W, A, B} & =                \id_A √ó œë‚Ä≥_{W, A, B}                          \\[\topsep]
œë_{W, A, B}  & \relwithsizeof=: A ‚ñ∑_W (A ‚ñ∑_W B) ‚Üí A ‚ñ∑_W B                     \notag\\
\label{equation:no-prime-vartheta}
œë_{W, A, B}  & =                \left[\id_{A ‚ñ∑_W B}, Œπ‚ÇÇ ‚àò œë‚Ä≤_{W, A, B}\right]
\end{align}

conditions. Regarding $Œ∏‚Ä≥$, we require that for all $W ‚àà \Ob(ùí≤)$ and $B ‚àà

\extsubsection{process-merging-and-the-canonical-nonterminating-process}
              {Process Merging and the Canonical Nonterminating Process}

There are additional constraints on APCs, which ensure that an APC also models
two other operations: process merging and the construction of the sole value
of~$1 ‚ñ∑‚Ä≥‚ÇÅ 0$, which is called the canonical nonterminating process. We do not
discuss these additional constraints here, because they are not fundamentally
related to our extension of APCs. We just mention them in the definition of APCs
in the next subsection.

\extsubsection{abstract-process-categories-summary}{Summary}

Let us now state the complete definition of abstract process categories.

\begin{extdefinition}{abstract-process-category}[Abstract process category]
Let $ùí≤$ be a category with finite products, and let $ùíû$ be a CCC with finite
coproducts. Furthermore let $‚ñ∑‚Ä≥$ be a functor from~$ùí≤ √ó ùíû √ó ùíû$ to~$ùíû$, let $‚ñ∑‚Ä≤$
and~$‚ñ∑$ be defined as in \eqref{equation:derived-process-functors}, and let $Œ∏‚Ä≥$
and~$œë‚Ä≥$ be natural transformations with the following typings:
\begin{align*}

\ref{figure:coherence-of-process-joining}

$Œ∏$, $œë‚Ä≤$, and~$œë$ are defined as in \eqref{equation:no-prime-theta},
\eqref{equation:single-prime-vartheta}, and~\eqref{equation:no-prime-vartheta}.

\item

The natural transformation $\left‚ü®œá‚Ä≥‚ÇÅ, œá‚Ä≥‚ÇÇ\right‚ü©$ with
\begin{align}

The third and fourth of the above propositions refer to process merging and the
canonical nonterminating process mentioned in

\extsection{apcs-with-recursion-and-corecursion}
           {APCs with Recursion and Corecursion}

In \subsectionref{process-expansion-and-joining}, we saw that every APC
comprises ideal comonads and ideal monads based on temporal functors. There
exists a special kind of ideal comonads, called recursive comonads, which
captures a form of comonadic recursion. Likewise there exists a special kind of
ideal monads, called completely iterative monads, which captures a form of
monadic corecursion. In this section, we introduce APCs with recursion and
corecursion (‚Ñõ-APCs), which comprise recursive comonads and completely iterative
monads that model recursion and corecursion on processes.

\extsubsection{corecursion-on-processes}{Corecursion on Processes}

If $(ùí≤, ùíû, ‚ñ∑‚Ä≥, Œ∏‚Ä≥, œë‚Ä≥)$ is an APC, then every pair $\left(A ‚ñ∑‚Ä≤_W {-}, œë‚Ä≤_{W, A,
{-}}\right)$ is an ideal monad. We extend the structure of APCs by requiring
that every such pair is a completely iterative monad. Completely iterative
monads are defined, for example, by Milius
\cite[Definition~5.5]{milius:ic-196-1}. We use a definition that is different
from the one by Milius, but nevertheless equivalent to it. We compare both
definitions in
\appendixref{changes-in-the-completely-iterative-monad-definition}, where we
also list the advantages that our definition has in our opinion.

\begin{extdefinition}{completely-iterative-monad}[Completely iterative monad]
Let $ùíû$ be a category with binary coproducts. A pair $(T‚Ä≤, Œº‚Ä≤)$ is a completely
iterative monad on~$ùíû$ if and only if it is an ideal monad on~$ùíû$, and for every
morphism $f : C ‚Üí T‚Ä≤(B + C)$, there exists a unique morphism $f^‚àû : C ‚Üí T‚Ä≤B$ for
which the diagram in
\figureref{condition-for-a-morphism-f-infinity-of-a-completely-iterative-monad}
commutes.
\end{extdefinition}
\begin{extfigure}{condition-for-a-morphism-f-infinity-of-a-completely-iterative-monad}
                 {Condition for a morphism~$f^‚àû$ of a completely iterative monad}

\begin{tikzpicture}[diagram,column sep=7em]

\node [object] (end)              {$T‚Ä≤B$};         \\

\edges

      (start)            edge [morphism] node [auto=right] {$f$}               (after expansion)
      (after expansion)  edge [morphism] node [auto=right] {$T‚Ä≤(\id_B + f^‚àû)$} (after inner call)
      (after inner call) edge [morphism] node [auto=right] {$Œº‚Ä≤_B$}            (end);

\end{tikzpicture}

By requiring that every pair $\left(A ‚ñ∑‚Ä≤_W {-}, œë‚Ä≤_{W, A, {-}}\right)$ is a
completely iterative monad, we ensure that for every morphism $f : C ‚Üí A ‚ñ∑‚Ä≤_W (B
+ C)$, there is a corresponding morphism $f^‚àû : C ‚Üí A ‚ñ∑‚Ä≤_W B$. Let $u$ and~$u^‚àû$
be the FRP operations that $f$ and~$f^‚àû$ model, and let $z$ be a value from the
domain of $u$ and~$u^‚àû$. The diagram in
\figureref{condition-for-a-morphism-f-infinity-of-a-completely-iterative-monad}
tells us how the process $u^‚àû(z)$ is defined:
\begin{itemize}

\item

If $u(z)$ terminates, and its terminal event carries a value $Œπ‚ÇÅ(y)$, then
$u^‚àû(z)$ terminates at the same time as $u(z)$, the continuous part of $u^‚àû(z)$
is the continuous part of $u(z)$, and the terminal event of $u^‚àû(z)$ carries the
value~$y$.

\item

If $u(z)$ terminates, and its terminal event carries a value $Œπ‚ÇÇ(z‚Ä≤)$, then
$u^‚àû(z)$ is the result of concatenating the continuous part of $u(z)$ and the
process~$u^‚àû(z‚Ä≤)$.

\item

If $u(z)$ does not terminate, then $u^‚àû(z)$ does not terminate, and the
continuous part of $u^‚àû(z)$ is the continuous part of $u(z)$.

\end{itemize}
Note that in the second case, $u^‚àû(z)$ is defined in terms of $u^‚àû(z‚Ä≤)$, but
only the suffix of $u^‚àû(z)$ that starts at the termination time of $u(z)$
actually depends on $u^‚àû(z‚Ä≤)$. The operator~${-}^‚àû$ models corecursion on
processes.

We derive a variant of~${-}^‚àû$ that works with~$‚ñ∑$ instead of~$‚ñ∑‚Ä≤$. We use the
notation ${-}^‚àû$ also for this variant. This overloading of notation is
possible, because we can always deduce from the types which variant of~${-}^‚àû$
is meant. The $‚ñ∑$-variant turns every morphism $f : C ‚Üí B + A ‚ñ∑‚Ä≤_W C$ into a
morphism~$f^‚àû$ that is defined as follows:
\begin{align}
f^‚àû & \relwithsizeof=: C ‚Üí A ‚ñ∑_W B                                                \notag\\
f^‚àû & =                \left(\id_B + \left(\id_A ‚ñ∑‚Ä≤_{\id_W} f\right)^‚àû\right) ‚àò f
\end{align}
Note that the ${-}^‚àû$ on the right-hand side refers to the original
$‚ñ∑‚Ä≤$-variant. There is also a variant of~${-}^‚àû$ that works with~$‚ñ∑‚Ä≥$. It turns
every morphism $f : C ‚Üí A ‚ñ∑‚Ä≥_W (B + A √ó C)$ into a morphism~$f^‚àû$ that is
defined as follows:
\begin{align}
f^‚àû & \relwithsizeof=: C ‚Üí A ‚ñ∑‚Ä≥_W B                                                             \notag\\
f^‚àû & =                œë‚Ä≥_{W, A, B} ‚àò \left(\id_A ‚ñ∑‚Ä≥_{\id_W} (\id_B + (\id_A √ó f)^‚àû)\right) ‚àò f
\end{align}
Again the ${-}^‚àû$ on the right-hand side refers to the original $‚ñ∑‚Ä≤$-variant.

\extsubsection{recursion-on-processes}{Recursion on Processes}

We saw in \subsectionref{process-expansion-and-joining} that every pair
$\left({-} ‚ñ∑‚Ä≥_W B, Œ∏‚Ä≥_{W, {-}, B}\right)$ is an ideal comonad. We extend the
structure of APCs by requiring that every such pair is a recursive comonad. The
notion of recursive comonad is dual to the notion of completely iterative monad.
So a recursive comonad on~$ùíû$ is just a completely iterative monad on~$\op{ùíû}$.
We give an explicit definition nevertheless.

\begin{extdefinition}{recursive-comonad}[Recursive comonad]
Let $ùíû$ be a category with binary products. A pair $(U‚Ä≤, Œ¥‚Ä≤)$ is a recursive
comonad on~$ùíû$ if and only if it is an ideal comonad on~$ùíû$, and for every
morphism $f : U‚Ä≤(A √ó C) ‚Üí C$, there exists a unique morphism $f^* : U‚Ä≤A ‚Üí C$ for
which the diagram in
\figureref{condition-for-a-morphism-f-star-of-a-recursive-comonad} commutes.
\end{extdefinition}

\begin{tikzpicture}[diagram,column sep=7em]

By requiring that every pair $\left({-} ‚ñ∑‚Ä≥_W B, Œ∏‚Ä≥_{W, {-}, B}\right)$ is a
recursive comonad, we ensure that for every morphism $f : (A √ó C) ‚ñ∑‚Ä≥_W B ‚Üí C$,
there is a corresponding morphism $f^* : A ‚ñ∑‚Ä≥_W B ‚Üí C$. Let $u$ and~$u^*$ be the
FRP operations that $f$ and~$f^*$ model, and let $p$ be a process from the
domain of~$u^*$. The diagram in
\figureref{condition-for-a-morphism-f-star-of-a-recursive-comonad} tells us that
the value $u^*(p)$ is $u\left(p^‚Ä†\right)$ where $p^‚Ä†$ is defined by the
following statements:
\begin{itemize}

\item

The process~$p^‚Ä†$ terminates if and only if~$p$ terminates.

\item

The terminal event of~$p^‚Ä†$, if any, is the terminal event of~$p$.

\item

The value of the continuous part of~$p^‚Ä†$ at a time~$t$ is $(x, u^*(p‚Ä≤))$ where
$x$ is the value of the continuous part of~$p$ at~$t$, and $p‚Ä≤$ is the suffix
of~$p$ that follows~$t$.

\end{itemize}
Note that $u^*(p)$ is defined in terms of $u^*(p‚Ä≤)$, but $p‚Ä≤$ is a proper suffix
of~$p$. The operator~${-}^*$ models recursion on processes.

We derive a variant of~${-}^*$ that works with~$‚ñ∑‚Ä≤$ instead of~$‚ñ∑‚Ä≥$. We use the
notation ${-}^*$ also for this variant. The $‚ñ∑‚Ä≤$-variant turns every morphism $f
: A √ó C ‚ñ∑‚Ä≥_W B ‚Üí C$ into a morphism~$f^*$ that is defined as follows:
\begin{align}
f^* & \relwithsizeof=: A ‚ñ∑‚Ä≤_W B ‚Üí C                                               \notag\\
f^* & =                f ‚àò \left(\id_A √ó \left(f ‚ñ∑‚Ä≥_{\id_W} \id_B\right)^*\right)
\end{align}
Note that this construction is dual to the construction of the $‚ñ∑$-variant
of~${-}^‚àû$.

\extsubsection{apcs-with-recursion-and-corecursion-summary}{Summary}

We formulate the definition of ‚Ñõ-APCs based on the above explanations.

\begin{extdefinition}{apc-with-recursion-and-corecursion}
                     [APC with recursion and corecursion]
An APC with recursion and corecursion (‚Ñõ-APC) is an APC $(ùí≤, ùíû, ‚ñ∑‚Ä≥, Œ∏‚Ä≥, œë‚Ä≥)$ for
which the following holds:
\begin{itemize}

\item

For all $W ‚àà \Ob(ùí≤)$ and $B ‚àà \Ob(ùíû)$, $\left({-} ‚ñ∑‚Ä≥_W B, Œ∏‚Ä≥_{W, {-}, B}\right)$
is a recursive comonad.

\item

For all $W ‚àà \Ob(ùí≤)$ and $A ‚àà \Ob(ùíû)$, $\left(A ‚ñ∑‚Ä≤_W {-}, œë‚Ä≤_{W, A, {-}}\right)$
is a completely iterative monad.

\end{itemize}
\end{extdefinition}

\extsection{concrete-process-categories}{Concrete Process Categories}

Concrete process categories (CPCs) are models of FRP with processes that use
concrete categorical constructions to express time-dependence of type
inhabitance and causality of FRP operations. They are described in detail in an
earlier publication of ours \cite[Section~3]{jeltsch:plpv-2013}. Here we only
give a short introduction to CPCs.

\extsubsection{core-structure}{Core Structure}

Let $(T, ‚â§)$ be a totally ordered set, and let $‚Ñ¨$ be a CCC with finite
coproducts. We use $(T, ‚â§)$ to model the time scale, and $‚Ñ¨$ to model ordinary
types and functions. Based on $(T, ‚â§)$, we construct a category~$‚Ñê$, which we
call the temporal index category of $(T, ‚â§)$.

\begin{extdefinition}{temporal-index-category}[Temporal index category]
The temporal index category of a totally ordered set $(T, ‚â§)$ is the
category~$‚Ñê$ for which the following holds:
\begin{align}
\Ob(‚Ñê)                                                  &
= \set{(t, \obstime) ‚àà T √ó T}{t ‚â§ \obstime}             \\
\hom_‚Ñê\bigl((t‚Ä≤,\obstime‚Ä≤),(t, \obstime)\bigr)          &
= \begin{cases}
\left\{\left(t, \obstime, \obstime‚Ä≤\right)\right\} &
  \text{if $t = t‚Ä≤$ and $\obstime ‚â§ \obstime‚Ä≤$}      \\
  ‚àÖ                                                  &
  \text{otherwise}
\end{cases}
\end{align}
\end{extdefinition}

We model FRP types and FRP operations by the functor category~$‚Ñ¨^‚Ñê$. Let $A$ be
an object of~$‚Ñ¨^‚Ñê$ that models an FRP type~$œÑ$. For every object $(t, \obstime)$
of~$‚Ñê$, the object $A(t, \obstime)$ of~$‚Ñ¨$ deals with the FRP values that
inhabit~$œÑ$ at~$t$; it describes the type whose inhabitants give the information
we have about these FRP values at~$\obstime$. We call $\obstime$ the observation
time. For every morphism $\left(t, \obstime, \obstime‚Ä≤\right)$ of~$‚Ñê$, the
morphism $A\left(t, \obstime, \obstime‚Ä≤\right)$ models the function that turns
information we have at~$\obstime‚Ä≤$ into information we have at~$\obstime$ by
forgetting all information acquired after~$\obstime$

The definition of CPCs requires the category~$‚Ñ¨$ to have some additional
properties, which are necessary for modeling function types and process types in
FRP.

\begin{extdefinition}{concrete-process-category}[Concrete process category]
Let $(T, ‚â§)$ be a totally ordered set, let $‚Ñê$ be its temporal index category,
and let $‚Ñ¨$ be a CCC with finite coproducts that has all products and coproducts
of families indexed by intervals $(t, t‚Ä≤) ‚äÜ T$ and all ends of the form $‚à´_{t‚Ä≥ ‚àà
[t, t‚Ä≤]} B(t, t‚Ä≥)^{A\left(t, t‚Ä≥\right)}$ where $A, B ‚àà ‚Ñ¨^‚Ñê$. Then the tuple $(T,
‚â§, ‚Ñ¨)$ is a concrete process category (CPC). The actual category that $(T, ‚â§,
‚Ñ¨)$ denotes is the functor category~$‚Ñ¨^‚Ñê$.
\end{extdefinition}

\extsubsection{the-basic-temporal-functor}{The Basic Temporal Functor}

The basic temporal functor of a CPC \cite[Subsection~4.3]{jeltsch:plpv-2014}
models the weak basic process type constructor as well as process type
constructors that put upper bounds on termination times.

We define the totally ordered set $(T_‚àû, ‚â§_‚àû)$ such that $T_‚àû = T ‚à™ \{‚àû\}$ and
\begin{equation*}
t‚ÇÅ ‚â§_‚àû t‚ÇÇ ‚áî t‚ÇÅ ‚â§ t‚ÇÇ ‚à® t‚ÇÇ = ‚àû\enspace.
\end{equation*}
Let $ùí≤$ be the category of $(T_‚àû, ‚â§_‚àû)$. We use $ùí≤$ to model constraints
regarding termination. Thereby any object $\timebound ‚àà T$ stands for
termination at or before the time~$\timebound$, and the object~$‚àû$ stands for
the absence of any termination guarantees. We define the basic temporal functor
as follows.

\begin{extdefinition}{basic-temporal-functor-of-a-cpc}
                     [Basic temporal functor of a CPC]
Let $(T, ‚â§, ‚Ñ¨)$ be a CPC, and let $ùí≤$ be defined as above. The basic temporal
functor of~$(T, ‚â§, ‚Ñ¨)$ is the functor $‚ñ∑‚Ä≥ : ùí≤ √ó ‚Ñ¨^‚Ñê √ó ‚Ñ¨^‚Ñê ‚Üí ‚Ñ¨^‚Ñê$ with
\begin{equation}
\label{equation:process-information-objects}
\left(A ‚ñ∑‚Ä≥_{\timebound} B \right)(t, \obstime)
    = \begin{cases}
0                                                     & \text{if $\timebound < t$}            \\
      S_{\timebound}                                        & \text{if $t ‚â§ \timebound ‚â§ \obstime$} \\
      S_{\obstime} + ‚àè_{t‚Ä≤ ‚àà (t, \obstime]} A(t‚Ä≤, \obstime) & \text{if $\obstime <_‚àû \timebound$}
\end{cases}
\end{equation}
where for every~$t^* ‚àà T$, $S_{t^*}$ is defined as follows:
\begin{equation}
\label{equation:process-information-objects-helper}
S_{t^*} = ‚àê_{t‚Ä≤ ‚àà (t, t^*]} \left(\left(‚àè_{t‚Ä≥ ‚àà (t, t‚Ä≤)} A(t‚Ä≥, \obstime)\right) √ó B(t‚Ä≤, \obstime)\right)
\end{equation}
\end{extdefinition}

The above definition is actually incomplete. A complete definition would also
define the following:
\begin{itemize}

\item

morphisms $\left(A ‚ñ∑‚Ä≥_W B\right)i$ where $W ‚àà \Ob(ùí≤)$, $A, B ‚àà
\Ob\left(‚Ñ¨^‚Ñê\right)$, and $i ‚àà \Mor(‚Ñê)$

\item

morphisms $\left(f ‚ñ∑‚Ä≥_{\id_W} g\right)_I$ where $W ‚àà \Ob(ùí≤)$, $f, g ‚àà
\Mor\left(‚Ñ¨^‚Ñê\right)$, and $I ‚àà \Ob(‚Ñê)$

\item

morphisms $\left(\id_A ‚ñ∑‚Ä≥_w \id_B\right)_I$ where $w ‚àà \Mor(ùí≤)$, $A, B ‚àà
\Ob\left(‚Ñ¨^‚Ñê\right)$, and $I ‚àà \Ob(‚Ñê)$

\end{itemize}
However defining these things is a mostly mechanical task. In particular, the
definition of morphisms $\left(f ‚ñ∑‚Ä≥_{\id_W} g\right)_I$ can be derived from
\eqref{equation:process-information-objects}
and~\eqref{equation:process-information-objects-helper} by replacing $0$ with
$\id_0$ and object expressions of the form $C(t, \obstime)$ with morphism
expressions of the form $h_{(t, \obstime)}$.

\extsubsection{relationship-to-abstract-process-categories}
              {Relationship to Abstract Process Categories}

APCs generalize CPCs. This is expressed by the following theorem.

\begin{exttheorem}{apc-from-cpc}
Let $(T, ‚â§, ‚Ñ¨)$ be a CPC, let $‚Ñê$ be the temporal index category of $(T, ‚â§)$,
let $‚ñ∑‚Ä≥$ be the basic temporal functor of $(T, ‚â§, ‚Ñ¨)$, and let $ùí≤$ be the
category of the totally ordered set $(T_‚àû, ‚â§_‚àû)$ that is defined as described in
\subsectionref{the-basic-temporal-functor}. Then there exist natural
transformations $Œ∏‚Ä≥$ and~$œë‚Ä≥$ such that $\left(ùí≤, ‚Ñ¨^‚Ñê, ‚ñ∑‚Ä≥, Œ∏‚Ä≥, œë‚Ä≥\right)$ is an
APC.
\end{exttheorem}

Please see our earlier work \cite[proof of Theorem~10]{jeltsch:plpv-2014} for a
proof of this theorem.

\extsection{cpcs-with-recursion-and-corecursion}
           {CPCs with Recursion and Corecursion}


According to \theoremref{apc-from-cpc}, every CPC gives rise to an APC. However
there is no guarantee that this APC is also an ‚Ñõ-APC. In this section, we
develop CPCs with recursion and corecursion (‚Ñõ-CPCs). ‚Ñõ-CPCs are a special kind
of CPCs whose corresponding APCs are ‚Ñõ-APCs.

\extsubsection{enabling-corecursion-on-processes}
              {Enabling Corecursion on Processes}

Let $(T, ‚â§, ‚Ñ¨)$ be a CPC. Let furthermore $\left(ùí≤, ‚Ñ¨^‚Ñê, ‚ñ∑‚Ä≥, Œ∏‚Ä≥, œë‚Ä≥\right)$ be
the APC it induces according to \theoremref{apc-from-cpc}. This APC models
corecursion on processes if and only if for every morphism $f : C ‚Üí A
‚ñ∑‚Ä≤_{\timebound} (B + C)$, there exists a unique morphism~$f^‚àû$ for which the
following holds:
\begin{align}
f^‚àû & \relwithsizeof=: C ‚Üí A ‚ñ∑‚Ä≤_{\timebound} B                                  \notag\\
\label{equation:f-infinity-for-apc}
f^‚àû & =                œë‚Ä≤_{\timebound, A, B}                                  ‚àò
                       \left(\id_A ‚ñ∑‚Ä≤_{\id_{\timebound}} (\id_B + f^‚àû)\right) ‚àò
                       f
\end{align}
From the definition of~$‚ñ∑‚Ä≤$ in \eqref{equation:derived-process-functors} and the
definition of the basic temporal functor in
\subsectionref{the-basic-temporal-functor}, it follows that
\eqref{equation:f-infinity-for-apc} is true if and only if for all $t, \obstime
‚àà T$ with $t ‚â§ \obstime$, we have
\begin{equation}
\label{equation:f-infinity-element-definition}
f^‚àû_{(t, \obstime)} = \left(œë‚Ä≤_{\timebound, A, B}\right)_{(t, \obstime)} ‚àò
                      (\id_A √ó h)                                        ‚àò
                      f_{(t, \obstime)}
\end{equation}
where $h$ is defined as follows:
\begin{equation}
\label{equation:f-infinity-element-definition-helper}
h = \begin{cases}
\id_0                                                                                                &
        \text {if $\timebound < t$}                                                                      \\
    ‚àê_{t‚Ä≤ ‚àà (t, \timebound]} \left(\id √ó \left(\id + f^‚àû_{(t‚Ä≤, \obstime)}\right)\right)                  &
        \text {if $t ‚â§ \timebound ‚â§ \obstime$}                                                           \\[\medskipamount]
    \left(‚àê_{t‚Ä≤ ‚àà (t, \obstime]} \left(\id √ó \left(\id + f^‚àû_{(t‚Ä≤, \obstime)}\right)\right)\right) + \id &
        \text {if $\obstime <_‚àû \timebound$}
    \end{cases}
\end{equation}
Equations \eqref{equation:f-infinity-element-definition}
and~\eqref{equation:f-infinity-element-definition-helper} define $f^‚àû_{(t,
\obstime)}$ in terms of morphisms $f^‚àû_{(t‚Ä≤, \obstime)}$ with $t‚Ä≤ > t$. So if
the order~$‚â•$ on the set $\set{t ‚àà T}{t ‚â§ \obstime}$ is well-founded,
\eqref{equation:f-infinity-element-definition}
and~\eqref{equation:f-infinity-element-definition-helper} define $f^‚àû_{(t,
\obstime)}$ by well-founded recursion with recursion parameter~$t$. As a
consequence, there exists a unique $f^‚àû$ that fulfills
\eqref{equation:f-infinity-for-apc}.

Based on the above considerations, we require for an ‚Ñõ-CPC that for all $t ‚àà T$,
the order~$‚â•$ on $\set{t‚Ä≤ ‚àà T}{t‚Ä≤ ‚â§ t}$ is well-founded, which means that we
cannot have an infinite ascending sequence of times that all lie before a
certain time. This makes $œâ$-supertasks~\cite{laraudogoitia:supertasks}
impossible, which, for example, implies that the sequence of actions described
in Zeno‚Äôs paradox of Achilles and the tortoise cannot occur.

Note however that we still allow time scales that are quite different from the
discrete time scale $(‚Ñï, ‚â§)$. For example, $\bigset{z + 1/n}{z ‚àà ‚Ñ§ \mathrel‚àß n ‚àà
‚Ñï ‚àñ \{0\}}$ with the usual ordering of rational numbers is still a perfectly
acceptable time scale.

\extsubsection{enabling-recursion-on-processes}{Enabling Recursion on Processes}

The APC $\left(ùí≤, ‚Ñ¨^‚Ñê, ‚ñ∑‚Ä≥, Œ∏‚Ä≥, œë‚Ä≥\right)$ induced by a CPC $(T, ‚â§, ‚Ñ¨)$ models
recursion on processes if and only if for every morphism $f : (A √ó C)
‚ñ∑‚Ä≥_{\timebound} B ‚Üí C$, there exists a unique morphism~$f^*$ for which the
following holds:
\begin{align}
f^* & \relwithsizeof=: A ‚ñ∑‚Ä≥_{\timebound} B ‚Üí C                                  \notag\\
\label{equation:f-star-for-apc}
f^* & =                f                                                      ‚àò
                       \left((\id_A √ó f^*) ‚ñ∑‚Ä≥_{\id_{\timebound}} \id_B\right) ‚àò
                       Œ∏‚Ä≥_{\timebound, A, B}
\end{align}
Equation~\eqref{equation:f-star-for-apc} is true if and only if for all $t,
\obstime ‚àà T$ with $t ‚â§ \obstime$, we have
\begin{equation}
\label{equation:f-star-element-definition}
f^*_{(t, \obstime)} = f_{(t, \obstime)}                                  ‚àò
                      h                                                  ‚àò
                      \left(Œ∏‚Ä≥_{\timebound, A, B}\right)_{(t, \obstime)}
\end{equation}
where $h$ is defined as follows:
\begin{align}
h      & = \begin{cases}
\id_0                                                                                          &
               \text {if $\timebound < t$}                                                                \\
           ‚àê_{t‚Ä≤ ‚àà (t, \timebound]} s_{t‚Ä≤}                                                                &
               \text {if $t ‚â§ \timebound ‚â§ \obstime$}                                                     \\
           ‚àê_{t‚Ä≤ ‚àà (t, \obstime]} s_{t‚Ä≤} + ‚àè_{t‚Ä≤ ‚àà (t, \obstime]} \left(\id √ó f^*_{(t‚Ä≤, \obstime)}\right) &
               \text {if $\obstime <_‚àû \timebound$}
\end{cases}                                                                                       \\
\label{equation:last-f-star-element-definition-helper}
s_{t‚Ä≤} & = \left(‚àè_{t‚Ä≥ ‚àà (t, t‚Ä≤)} \left(\id √ó f^*_{(t‚Ä≥, \obstime)}\right)\right) √ó \id
\end{align}
Equations \eqref{equation:f-star-element-definition}
through~\eqref{equation:last-f-star-element-definition-helper} define $f^*_{(t,
\obstime)}$ by well-founded recursion, assuming that the order~$‚â•$ on the set
$\set{t ‚àà T}{t ‚â§ \obstime}$ is well-founded. As a consequence, there exists a
unique $f^*$ that fulfills \eqref{equation:f-star-for-apc}.

\extsubsection{cpcs-with-recursion-and-corecursion-summary}{Summary}

We formulate the definition of ‚Ñõ-CPCs based on the above explanations.

\begin{extdefinition}{cpc-with-recursion-and-corecursion}
                     [CPC with recursion and corecursion]
A CPC $(T, ‚â§, ‚Ñ¨)$ is a CPC with recursion and corecursion (‚Ñõ-CPC) if and only if
for all $t ‚àà T$, the order~$‚â•$ on $\set{t‚Ä≤ ‚àà T}{t‚Ä≤ ‚â§ t}$ is well-founded.
\end{extdefinition}

We have developed ‚Ñõ-CPCs such that they give rise to ‚Ñõ-APCs. We state the
relationship between ‚Ñõ-CPCs and ‚Ñõ-APCs in the following theorem.

\begin{exttheorem}{r-apc-from-r-cpc}
The APC that is induced by an ‚Ñõ-CPC according to \theoremref{apc-from-cpc} is an
‚Ñõ-APC.
\end{exttheorem}

\extsection{related-work}{Related Work}

We can build ‚Ñõ-APCs by taking the categorical semantics of the intuitionistic
modal logic IK~\cite{bellin:m4m-2001} and successively adding more structure.
This leads us first to intuitionistic S4 categories
\cite[Section~4]{jeltsch:entcs-286}, which are closely related to the
categorical semantics of intuitionistic S4 variants by
Kobayashi~\cite{kobayashi:tcs-175-1} and Bierman and
de~Paiva~\cite{bierman:sl-65-3}. Subsequent additions to intuitionistic S4
categories lead us to temporal categories \cite[Sections 5
and~6]{jeltsch:entcs-286}, from there to APCs~\cite{jeltsch:plpv-2014}, and
finally to ‚Ñõ-APCs.

Krishnaswami and Benton~\cite{krishnaswami:lics-2011} model FRP categorically
based on 1-bounded ultrametric spaces. Their semantics ensures that morphisms
only model causal functions, which is also a key property of CPCs. However they
model only behaviors, not events or even processes. Furthermore they require the
time scale to be discrete. Although this is a restriction, it makes it possible
to model recursion on behaviors.

Cave et~al.~\cite{cave:popl-2014} present an FRP calculus that features
inductive and coinductive types and a type constructor that corresponds to the
‚Äúnext‚Äù modality of LTL. The authors show that process types are definable within
their calculus. We strongly conjecture that their calculus also allows for the
definition of recursion and corecursion operators on processes. The downside of
their solution is that it is fundamentally tied to discrete time.

Jeffrey \cite[Section~2]{jeffrey:plpv-2012} defines categories ${‚äü}\rset$ and
${‚äµ}\rset$. The morphisms of ${‚äü}\rset$ model causal functions on time-varying
values, and the morphisms of ${‚äµ}\rset$ model causal functions that produce
output values based only on a finite history. Jeffrey also introduces a fixed
point operator $\ifix$, which is similar to our ${-}^*$ operator for recursion,
but uses time-varying values that lie in the past, while ${-}^*$ uses processes,
whose continuous parts lie in the future. Consequently Jeffrey requires that
$‚â§$, and not $‚â•$, is well-founded on certain intervals.

Birkedal et~al.~\cite{birkedal:lmcs-8-4} model guarded recursion by the topos of
trees. The topos of trees is the category $ùíÆ = \Set^{\op{ùí©}}$, where $ùí©$ is the
category of the ordered set $(‚Ñï, ‚â§)$. If $A ‚àà \Ob(ùíÆ)$ models a type~$œÑ$, an
object~$An$ with $n ‚àà ‚Ñï$ models the type whose inhabitants give the information
we have about inhabitants of~$œÑ$ after $n$ recursion steps, and a morphism~$Av$
with $v : m ‚Üí n$ models the function that forgets the information acquired
during recursion steps $n + 1$ through~$m$. There is a close connection between
this development and CPCs. Consider the CPC $(‚Ñï, ‚â§, \Set)$. Let $‚Ñê$ be the
temporal index category of this CPC, and let $‚Ñ∞$ be the full subcategory
of~$\Set^‚Ñê$ that is induced by those objects that model types with
time-independent inhabitance, that is, by those $A ‚àà \Ob\left(\Set^‚Ñê\right)$
with $‚àÄ k, l, n ‚àà ‚Ñï . A(k, k + n) = A(l, l + n)$ and a similar condition for
morphisms. Then the categories $ùíÆ$ and~$‚Ñ∞$ are isomorphic. Birkedal et~al.
define a fixed point operator based on a ‚Äúlater‚Äù type constructor that
corresponds to the ‚Äúnext‚Äù modality of temporal logic. Studying the relationship
between this fixed point operator and our structure for recursion and
corecursion on processes remains a task for the future.

\extsection{conclusions-and-further-work}{Conclusions and Further Work}

We have developed ‚Ñõ-APCs and ‚Ñõ-CPCs, which are categorical models of FRP with
recursion and corecursion on processes. ‚Ñõ-APCs are defined purely axiomatically,
while ‚Ñõ-CPCs use concrete structure to express time-dependence of type
inhabitance and causality of FRP operations. We have furthermore shown that
‚Ñõ-APCs generalize ‚Ñõ-CPCs.

In the future, we want to develop categorical models of FRP with support for
mutable state. Another goal of ours is to implement FRP in mainstream functional
programming languages such that the interface directly reflects the abstract
categorical semantics.

\extappendix{changes-in-the-completely-iterative-monad-definition}
            {Changes in the Completely Iterative Monad Definition}

Milius \cite[Definition~5.5]{milius:ic-196-1} defines completely iterative
monads differently from us. A simplified version of his definition is as
follows.

\begin{extdefinition}{milius-style-completely-iterative-monad}
                     [Milius-style completely iterative monad]
Let $ùíû$ be a category with binary coproducts. A pair $(T‚Ä≤, Œº‚Ä≤)$ is a
Milius-style completely iterative monad on~$ùíû$ if and only if it is an ideal
monad on~$ùíû$, and for every morphism $f : C ‚Üí B + T‚Ä≤(B + C)$, there exists a
unique morphism $f^‚Ä† : C ‚Üí TB$ with $T = \Id + T‚Ä≤$ for which the diagram in
\figureref{condition-for-a-morphism-f-dagger-of-a-milius-style-completely-iterative-monad}
commutes.
\end{extdefinition}
\begin{extfigure}{condition-for-a-morphism-f-dagger-of-a-milius-style-completely-iterative-monad}
                 {Condition for a morphism~$f^‚Ä†$ of a Milius-style completely iterative monad}

\begin{tikzpicture}[diagram,column sep=7em]

\matrix{
\node [object] (start)            {$C$};             &
\node [object] (end)              {$TB$};            \\
\node [object] (after expansion)  {$B + T‚Ä≤(B + C)$}; &
\node [object] (after inner call) {$B + T‚Ä≤TB$};      \\
};

\edges

\path (start)            edge [morphism] node [auto=left]  {$f^‚Ä†$}                            (end)
      (start)            edge [morphism] node [auto=right] {$f$}                              (after expansion)
      (after expansion)  edge [morphism] node [auto=right] {$\id_B + T‚Ä≤\left[Œπ‚ÇÅ, f^‚Ä†\right]$} (after inner call)
      (after inner call) edge [morphism] node [auto=right] {$\id_B + Œº‚Ä≤_B$}                   (end);

\end{tikzpicture}

\end{extfigure}

We do not use this definition in this paper, because in our opinion, it has
disadvantages:
\begin{itemize}

\item

The morphism~$f : C ‚Üí B + T‚Ä≤(B + C)$ gives us two points per iteration where
iterating can come to an end. These two points correspond to the two sums in the
codomain of~$f$. However only one such point is necessary.


\item

The codomain of~$f^‚Ä†$ uses the derived functor~$T$ instead of the actual ideal
monad functor~$T‚Ä≤$, weakening the type of~$f^‚Ä†$.

\end{itemize}
However our own definition is equivalent to the one by Milius. This fact is
expressed by the following two theorems.

\begin{exttheorem}{jeltsch-cim-is-milius-cim}
If $(T‚Ä≤, Œº‚Ä≤)$ is a completely iterative monad on a category~$ùíû$ according to
\definitionref{completely-iterative-monad}, then it is a Milius-style completely
iterative monad on~$ùíû$.
\end{exttheorem}

\begin{extproof}
Let $(T‚Ä≤, Œº‚Ä≤)$ be a completely iterative monad according to
\definitionref{completely-iterative-monad}, and let $g$ be an arbitrary morphism
with $g : C ‚Üí B + T‚Ä≤(B + C)$. We have to show that there exists a unique
morphism~$g^‚Ä†$ for which the following holds:
\begin{align}
g^‚Ä† & \relwithsizeof=: C ‚Üí B + T‚Ä≤B                                                         \notag\\
g^‚Ä† & =                \left(\id_B + \left(Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right]\right)\right) ‚àò g
\end{align}
We define $f$ as follows:
\begin{align}
f & \relwithsizeof=: T‚Ä≤(B + C) ‚Üí T‚Ä≤(B + T‚Ä≤(B + C)) \notag\\
\label{equation:f-in-equivalence-proof-1}
f & =                T‚Ä≤[Œπ‚ÇÅ, g]
\end{align}
Since $(T‚Ä≤, Œº')$ is a completely iterative monad according to
\definitionref{completely-iterative-monad}, there exists a unique $f^‚àû$ for
which the following holds:
\begin{align}
f^‚àû & \relwithsizeof=: T‚Ä≤(B + C) ‚Üí T‚Ä≤B            \notag\\
\label{equation:f-infinity-in-equivalence-proof-1}
f^‚àû & =                Œº‚Ä≤_B ‚àò T‚Ä≤(\id_B + f^‚àû) ‚àò f
\end{align}
According to \eqref{equation:f-in-equivalence-proof-1}, we have
\begin{equation}
\begin{split}
Œº‚Ä≤_B ‚àò T‚Ä≤(\id_B + f^‚àû) ‚àò f & = Œº‚Ä≤_B ‚àò T‚Ä≤(\id_B + f^‚àû) ‚àò T‚Ä≤[Œπ‚ÇÅ, g]        \\
                           & = Œº‚Ä≤_B ‚àò T‚Ä≤[Œπ‚ÇÅ, (\id_B + f^‚àû) ‚àò g]\enspace,
\end{split}
\end{equation}
so \eqref{equation:f-infinity-in-equivalence-proof-1} is equivalent to the
following equation:
\begin{equation}
f^‚àû = Œº‚Ä≤_B ‚àò T‚Ä≤[Œπ‚ÇÅ, (\id_B + f^‚àû) ‚àò g]
\end{equation}
We define the functions $q$ and~$r$ on morphisms as follows:
\begin{align}
& q \relwithsizeof=: \Hom(T‚Ä≤(B + C), T‚Ä≤B) ‚Üí \Hom(C, B + T‚Ä≤B) \notag\\
& q(a) =             (\id_B + a) ‚àò g                      \\[\topsep]
& r \relwithsizeof=: \Hom(C, B + T‚Ä≤B) ‚Üí \Hom(T‚Ä≤(B + C), T‚Ä≤B) \notag\\
& r(b) =             Œº‚Ä≤_B ‚àò T‚Ä≤[Œπ‚ÇÅ, b]
\end{align}
We know that there exists a unique $f^‚àû$ with
\begin{equation}
\label{equation:f-infinity-simplified-in-equivalence-proof-1}
f^‚àû = r(q(f^‚àû))\enspace,
\end{equation}
and we have to prove that there exists a unique $g^‚Ä†$ with
\begin{equation}
\label{equation:g-dagger-simplified-in-equivalence-proof-1}
g^‚Ä† = q\left(r\left(g^‚Ä†\right)\right)\enspace.
\end{equation}
From \eqref{equation:f-infinity-simplified-in-equivalence-proof-1}, it follows
that
\begin{equation}
q(f^‚àû) = q(r(q(f^‚àû)))\enspace,
\end{equation}
so $g^‚Ä† = q(f^‚àû)$ fulfills
\eqref{equation:g-dagger-simplified-in-equivalence-proof-1}. On the other hand,
if we have a $g^‚Ä†$ that fulfills
\eqref{equation:g-dagger-simplified-in-equivalence-proof-1}, we know that
\begin{equation}
r\left(g^‚Ä†\right) = r\left(q\left(r\left(g^‚Ä†\right)\right)\right)\enspace.
\end{equation}
Because there is only one $f^‚àû$ that fulfills
\eqref{equation:f-infinity-simplified-in-equivalence-proof-1}, it follows that
$f^‚àû = r\left(g^‚Ä†\right)$. From this, we get
\begin{equation}
g^‚Ä† = q\left(r\left(g^‚Ä†\right)\right) = q(f^‚àû)\enspace.
\end{equation}
So $g^‚Ä† = q(f^‚àû)$ is the unique solution of
\eqref{equation:g-dagger-simplified-in-equivalence-proof-1}.
\end{extproof}

\begin{exttheorem}{milius-cim-is-jeltsch-cim}
If $(T‚Ä≤, Œº‚Ä≤)$ is a Milius-style completely iterative monad on a category~$ùíû$,
then it is a completely iterative monad on~$ùíû$ according to
\definitionref{completely-iterative-monad}.
\end{exttheorem}

\begin{extproof}
Let $(T‚Ä≤, Œº‚Ä≤)$ be a Milius-style completely iterative monad, and let $f$ be an
arbitrary morphism with $f : C ‚Üí T‚Ä≤(B + C)$. We have to show that there exists a
unique morphism~$f^‚àû$ for which the following holds:
\begin{align}
f^‚àû & \relwithsizeof=: C ‚Üí T‚Ä≤B                     \notag\\
f^‚àû & =                Œº‚Ä≤_B ‚àò T‚Ä≤[Œπ‚ÇÅ, Œπ‚ÇÇ ‚àò f^‚àû] ‚àò f
\end{align}
We define $g$ as follows:
\begin{align}
g & \relwithsizeof=: C ‚Üí B + T‚Ä≤(B + C) \notag\\
\label{equation:g-in-equivalence-proof-2}
g & =                Œπ‚ÇÇ ‚àò f
\end{align}
Since $(T‚Ä≤, Œº')$ is a Milius-style completely iterative monad, there exists a
unique $g^‚Ä†$ for which the following holds:
\begin{align}
g^‚Ä† & \relwithsizeof=: C ‚Üí B + T‚Ä≤B                                                         \notag\\
\label{equation:g-dagger-in-equivalence-proof-2}
g^‚Ä† & =                \left(\id_B + \left(Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right]\right)\right) ‚àò g
\end{align}
According to \eqref{equation:g-in-equivalence-proof-2}, we have
\begin{equation}
\begin{split}
\left(\id_B + \left(Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right]\right)\right) ‚àò g
    & = \left(\id_B + \left(Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right]\right)\right) ‚àò Œπ‚ÇÇ ‚àò f \\
    & = Œπ‚ÇÇ ‚àò Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right] ‚àò f\enspace,
\end{split}
\end{equation}
so \eqref{equation:g-dagger-in-equivalence-proof-2} is equivalent to the
following equation:
\begin{equation}
g^‚Ä† = Œπ‚ÇÇ ‚àò Œº‚Ä≤_B ‚àò T‚Ä≤\left[Œπ‚ÇÅ, g^‚Ä†\right] ‚àò f
\end{equation}
We define the functions $q$ and~$r$ on morphisms as follows:
\begin{align}
& q \relwithsizeof=: \Hom(C, B + T‚Ä≤B) ‚Üí \Hom(C, T‚Ä≤B) \notag\\
& q(a) =             Œº‚Ä≤_B ‚àò T‚Ä≤[Œπ‚ÇÅ, a] ‚àò f            \\[\topsep]
& r \relwithsizeof=: \Hom(C, T‚Ä≤B) ‚Üí \Hom(C, B + T‚Ä≤B) \notag\\
& r(b) =             Œπ‚ÇÇ ‚àò b
\end{align}
We know that there exists a unique $g^‚Ä†$ with
\begin{equation}
g^‚Ä† = r\left(q\left(g^‚Ä†\right)\right)\enspace,
\end{equation}
and we have to prove that there exists a unique $f^‚àû$ with
\begin{equation}
\label{equation:f-infinity-simplified-in-equivalence-proof-2}
f^‚àû = q(r(f^‚àû))\enspace.
\end{equation}
